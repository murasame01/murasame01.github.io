虚函数

静态编联: 编译期间就决定了程序运行时将具体调用哪个函数体, 即使没有主程序也能知道程序各个函数之间的调用关系

动态编联: 在运行期间决定调用哪个函数体

实现方式之一 (虚机制: 使用虚拟函数和虚拟函数表)

虚函数的声明与定义

声明: virtual 返回类型 函数名 (参数列表) [const];

定义: 同一般成员函数

  1. 必须是成员函数

  2. 静态成员函数和构造函数, 拷贝构造函数不能是虚的

  3. 析构函数可以是虚的

  4. 若类中有其它虚函数 那么析构函数也应该是虚的

  5. 赋值函数通常不定义为虚的

  6. 通常采用 public 继承方式

  7. 继承自基类的虚函数(除虚的析构函数)

  8. 若基类的析构函数是虚的, 那么派生类中的析构函数也是虚的

虚函数表

一个指针数组, 各元素存放对应虚函数的入口地址

  • 要求对应的类至少有一个虚函数

  • 一个类至多有一个虚函数表, 同一个类的不同对象共享该表

  • 首次创建该类实例对象时在内存中同时创建该类的虚拟表

  • 按照函数顺序的序号依次存放入口地址

函数调用的编译

已 p -> Func() 为例

  • 确定 p 的静态类型, 如 A*

  • 在 A 类中寻找名字为 Func, 且参数可以匹配的函数

  • 若找不到, 编译错误

  • 若找到 判断该函数是否为 virtual 函数

  • 若不是 编译成 p->Func();

  • 若是虚函数 采用动态联编 从而编译成 (_p -> vptr) [index] ( (void _) p ,…); 即在运行时 根据 vptr 中的函数入口地址 选择执行函数

若希望 pObj->Func(); 或 obj.Func() 合法 必须有:

  • pObj / obj 的静态类型中必须有匹配的函数 Func;

  • 即使匹配的 Func 永远不被调用也要有

虚函数的访问

  • 虚函数中访问的非虚函数: 静态联编, 使用本地版本

  • 非虚函数中访问的虚函数: 动态联编

  • 虚函数中访问虚函数: 动态联编

  • 构造函数和虚函数

    • 构造函数不能为虚函数
    • 调用的虚函数采用静态联编 使用本地版本

  • 析构函数和虚函数

    • 析构函数可以为虚函数

    • 若类中有虚函数 那么析构函数也因为虚函数

    • 调用的虚函数采用静态联编 使用本地版本

Run Time Type Identify

  • typeid(exp) 或 typeid(type_name), 返回 const type_info 对象

    • exp 是指针 则返回静态类型对应的

    • exp 是类类型(对象/引用) 且含有虚函数 则返回动态类型对应的

  • dynamic_cast(exp)

    • T 只能是指针或引用
    • 若 T 为指针 成功则返回 T 类型指针; 失败返回 nullptr
    • 若 T 为引用 成功则返回 T 类型引用; 失败产生 bad_cast 异常

虚机制的意义和不足

  • 使用虚机制使得当更变子类的具体实现时 不用改客户端类的定义及实现

  • 只使用虚机制 难以适应多个方向的变化

    通常使用 水平关系(关联 依赖) + 继承
